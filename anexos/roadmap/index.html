    Roadmap de Evolução do Projeto

    Este documento descreve uma visão de alto nível para a evolução futura do sistema SisCoE, com foco em melhorias de arquitetura, automação e funcionalidades.

    Curto Prazo (Próximos 3-6 meses)

    • [CI/CD] Implementação do Pipeline de CI:

      • Objetivo: Automatizar a execução de testes e a análise de código a cada commit.
      • Ações: Configurar o GitHub Actions para rodar os testes do Django e um linter (como Ruff) em cada Pull Request.

    • [Monitoramento] Implementação de Rastreamento de Erros:

      • Objetivo: Capturar e analisar erros de produção em tempo real.
      • Ações: Integrar o Sentry ou GlitchTip à aplicação Django para monitoramento de exceções.

    • [Testes] Aumento da Cobertura de Testes:

      • Objetivo: Aumentar a confiança nas alterações de código.
      • Ações: Focar em escrever testes unitários e de integração para os apps mais críticos, como efetivo e accounts, visando uma cobertura de testes acima de 80%.

    Médio Prazo (6-12 meses)

    • [Arquitetura] Desacoplamento do Frontend (SPA):

      • Objetivo: Modernizar a arquitetura, desacoplar o frontend do backend e melhorar a experiência do usuário.
      • Ações:
        1. Formalizar a API interna em uma API REST completa usando o Django REST Framework.
        2. Iniciar o desenvolvimento de um novo frontend como uma Single Page Application (SPA), utilizando um framework moderno como React, Vue.js ou Svelte.

    • [Deploy] Implementação do Pipeline de CD:

      • Objetivo: Automatizar o deploy da aplicação em produção.
      • Ações: Configurar o GitHub Actions para fazer o deploy automático da aplicação no servidor de produção após um merge bem-sucedido na branch main.

    • [Monitoramento] Implementação de Métricas e Dashboards:

      • Objetivo: Obter visibilidade sobre a performance e a saúde da aplicação.
      • Ações: Integrar o django-prometheus para expor métricas da aplicação e configurar dashboards no Grafana para visualizar tempo de resposta, taxa de erros, etc.

    Longo Prazo (12+ meses)

    • [API] Evolução para uma API Pública:

      • Objetivo: Permitir a integração do SisCoE com outros sistemas.
      • Ações: Documentar e publicar a API REST, implementando autenticação baseada em tokens (como OAuth2) para clientes externos.

    • [Arquitetura] Refatoração para Microsserviços (se necessário):

      • Objetivo: Melhorar a escalabilidade e a manutenibilidade de módulos complexos.
      • Ações: Avaliar se algum dos apps Django (como efetivo ou relatorios) se beneficiaria de ser extraído do monólito para um microsserviço independente, com seu próprio banco de dados e ciclo de deploy.

    • [Segurança] Auditoria de Segurança Completa:

      • Objetivo: Garantir o mais alto nível de segurança para os dados.
      • Ações: Contratar uma auditoria de segurança externa ou realizar uma revisão de segurança interna aprofundada, incluindo testes de penetração (pentest).

    Este roadmap é um guia flexível e deve ser revisado e adaptado periodicamente com base nas necessidades do projeto e no feedback dos usuários.
