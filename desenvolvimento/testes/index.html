    Testes

    A estratégia de testes no SisCoE segue as convenções do framework Django, utilizando o unittest do Python como base. Os testes são fundamentais para garantir a qualidade, a estabilidade e a manutenibilidade do código.

    Framework de Testes

    O projeto utiliza o framework de testes integrado do Django, que fornece um conjunto de ferramentas e utilitários para simular requisições, interagir com o banco de dados de teste e verificar o comportamento da aplicação.

    Localização dos Testes

    Os testes são organizados por app. Cada app Django (accounts, efetivo, etc.) possui um arquivo tests.py, onde os casos de teste para aquele app específico devem ser escritos.

    Manter os testes junto ao código do app facilita a localização e a execução de testes relevantes a uma determinada funcionalidade.

    Como Executar os Testes

    Para executar o conjunto completo de testes da aplicação, utilize o seguinte comando do manage.py:

    python manage.py test

    Este comando descobrirá e executará automaticamente todos os testes definidos nos arquivos tests.py de todos os apps do projeto.

    Para executar os testes de um app específico, você pode especificar o nome do app:

    python manage.py test backend.efetivo

    Escrevendo Novos Testes

    Ao desenvolver novas funcionalidades ou corrigir bugs, é crucial adicionar testes que cubram as alterações.

    • Testes de Modelo: Verificam se a lógica customizada nos modelos (properties, methods) funciona como esperado.
    • Testes de View: Simulam requisições HTTP (GET, POST) para as views e verificam se a resposta está correta (código de status, conteúdo do template, redirecionamentos).
    • Testes de Formulário: Verificam a validação e o processamento de formulários.

    Exemplo de um teste simples em tests.py:

    from django.test import TestCase
from django.contrib.auth import get_user_model

User = get_user_model()

class AccountsTestCase(TestCase):

    def setUp(self):
        # Cria um usuário para ser usado nos testes
        self.user = User.objects.create_user(
            email='testuser@example.com',
            password='testpassword123',
            first_name='Test',
            last_name='User'
        )

    def test_user_creation(self):
        """Verifica se o usuário foi criado corretamente."""
        self.assertEqual(self.user.email, 'testuser@example.com')
        self.assertTrue(self.user.is_active)
        self.assertFalse(self.user.is_staff)

    def test_login_view(self):
        """Verifica se a página de login responde corretamente."""
        response = self.client.get('/accounts/login/')
        self.assertEqual(response.status_code, 200)
        self.assertTemplateUsed(response, 'accounts/login.html')

    Manter uma boa cobertura de testes é essencial para a saúde do projeto a longo prazo, permitindo que refatorações e novas implementações sejam feitas com mais segurança.
