    Configuração do Ambiente de Desenvolvimento

    Este guia descreve os passos necessários para configurar o ambiente de desenvolvimento do SisCoE em uma máquina local. A configuração utiliza Docker para os serviços de banco de dados e Python venv para o ambiente da aplicação.

    Pré-requisitos

    • Git
    • Python 3.12 ou superior
    • Docker e Docker Compose

    1. Clonar o Repositório

    Primeiro, clone o repositório do projeto a partir do GitHub:

    git clone git@github.com:Andre-fdias/SisCoE.git
cd SisCoE

    2. Configurar o Ambiente Virtual Python

    Para isolar as dependências do projeto, crie um ambiente virtual:

    python3 -m venv venv

    Ative o ambiente virtual:

    No Linux/macOS: 

    source venv/bin/activate

    No Windows: 

    .\venv\Scripts\activate

    3. Instalar as Dependências

    Com o ambiente virtual ativado, instale as dependências Python listadas no arquivo requirements.txt:

    pip install -r requirements.txt
    Nota: O arquivo requirements.txt no repositório pode conter pacotes com nomes incorretos (redis-6.4.0, brevo-python-1.2.0). Eles foram corrigidos para redis e brevo-python para que a instalação funcione.

    4. Configurar Variáveis de Ambiente

    O projeto utiliza um arquivo .env para gerenciar variáveis de ambiente. O docker-compose.yml espera um arquivo .env.docker.

    Crie um arquivo .env.docker na raiz do projeto. Você pode copiar o arquivo contrib/env_gen.py (se existir um template) ou criá-lo do zero com as seguintes variáveis essenciais:

    # Chave secreta do Django (gere uma nova)
SECRET_KEY='sua-chave-secreta-aqui'

# Configurações de Debug
DEBUG=True

# Hosts permitidos
ALLOWED_HOSTS=localhost,127.0.0.1

# Chaves de API (deixe em branco se não for usar as integrações)
BREVO_API_KEY=
GROQ_API_KEY=
WEATHER_API_KEY=

    5. Iniciar os Serviços com Docker

    O docker-compose.yml gerencia os serviços de banco de dados (PostgreSQL), PgAdmin e Mailhog. Para iniciá-los, execute:

    docker-compose up -d
    Isso iniciará os contêineres em segundo plano.

    6. Executar as Migrações do Banco

    Com o banco de dados rodando no Docker, aplique as migrações do Django para criar as tabelas do banco de dados:

    python manage.py migrate

    7. Criar um Superusuário

    Para acessar a área administrativa do Django, crie um superusuário:

    python manage.py createsuperuser
    Siga as instruções para definir um e-mail e senha.

    8. Iniciar o Servidor de Desenvolvimento

    Finalmente, inicie o servidor de desenvolvimento do Django:

    python manage.py runserver

    A aplicação estará disponível em http://127.0.0.1:8000.
