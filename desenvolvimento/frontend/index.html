    Desenvolvimento Frontend

    O frontend do SisCoE é construído com uma abordagem tradicional de renderização no lado do servidor (Server-Side Rendering) utilizando o sistema de templates do Django. Não se trata de uma Single Page Application (SPA).

    Sistema de Templates Django

    A interface do usuário é gerada a partir de templates HTML que são processados pelo backend do Django.

    • Localização: Os templates estão localizados dentro do subdiretório templates/ de cada app Django (ex: backend/core/templates/core/).
    • Herança: O sistema utiliza a herança de templates do Django. Geralmente, existe um template base (base.html) que define a estrutura principal da página (cabeçalho, rodapé, menu de navegação), e os templates específicos de cada página estendem este base, preenchendo os blocos de conteúdo.
    • Tags e Filtros: São utilizadas as tags e filtros padrão do Django ({% for %}, {{ variable }}, etc.), além de tags customizadas que podem ser encontradas nos diretórios templatetags/ de alguns apps.

    Estilização com TailwindCSS

    A estilização da aplicação é feita utilizando o framework TailwindCSS.

    • Configuração: O arquivo tailwind.config.js na raiz do projeto define as configurações do Tailwind, como tema, cores e plugins.
    • Compilação: O TailwindCSS funciona como uma ferramenta de desenvolvimento. As classes de utilidade usadas nos templates HTML são escaneadas, e um arquivo CSS final é gerado com apenas o CSS necessário. O comando para compilar o CSS geralmente é configurado no package.json.
    • Fluxo de Trabalho: Para adicionar ou modificar estilos, o desenvolvedor deve usar as classes de utilidade do Tailwind diretamente no HTML dos templates. Após a alteração, o processo de compilação do CSS precisa ser executado para que as mudanças sejam refletidas.

    JavaScript e Interatividade

    A interatividade no frontend é adicionada através de bibliotecas JavaScript. As principais dependências de frontend são gerenciadas via npm e estão listadas no package.json.

    • package.json: 

      "dependencies": {
  "@fullcalendar/core": "^6.1.15",
  "@fullcalendar/daygrid": "^6.1.15",
  "@fullcalendar/google-calendar": "^6.1.15",
  "@fullcalendar/list": "^6.1.15",
  "@fullcalendar/timegrid": "^6.1.15",
  "@tailwindcss/vite": "^4.0.9",
  "flowbite": "^3.1.2"
}

    • Principais Bibliotecas:

      • Flowbite: Uma biblioteca de componentes de UI construída sobre o TailwindCSS, usada para elementos interativos como modais, dropdowns e tooltips.
      • FullCalendar: Utilizada na aplicação agenda para criar uma interface de calendário rica e interativa.

    • Arquivos Estáticos: Os arquivos JavaScript e CSS compilados, juntamente com as imagens, estão localizados nos diretórios static/ de cada app. O Django os coleta para um diretório único durante o deploy.
