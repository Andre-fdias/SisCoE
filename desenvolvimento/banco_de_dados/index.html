    Banco de Dados

    O SisCoE utiliza um banco de dados relacional para persistir todos os dados da aplicação. A interação com o banco é gerenciada quase que exclusivamente pelo ORM (Object-Relational Mapper) do Django.

    Sistema de Gerenciamento de Banco de Dados (SGBD)

    • Produção: O ambiente de produção, orquestrado com Docker, utiliza o PostgreSQL, um SGBD de código aberto robusto e altamente escalável.
    • Desenvolvimento: O arquivo settings.py padrão está configurado para usar SQLite.

    ⚠️ Recomendação: Para garantir a paridade entre os ambientes e evitar bugs que só aparecem em produção, é fortemente recomendado configurar o ambiente de desenvolvimento para também utilizar o PostgreSQL através do Docker.

    ORM do Django e Models

    O Django possui um ORM poderoso que abstrai as interações com o banco de dados. Em vez de escrever SQL, os desenvolvedores definem Modelos (classes Python) que o Django traduz para tabelas e consultas SQL.

    • Definição de Modelos: Os modelos são definidos no arquivo models.py de cada app. Cada classe de modelo representa uma tabela no banco de dados, e cada atributo da classe representa uma coluna.

    Exemplo (efetivo/models.py): 

    class Cadastro(models.Model):
    id = models.AutoField(primary_key=True)
    re = models.CharField(max_length=6,  blank=False, null=False, unique=True)
    dig = models.CharField(max_length=1, blank=False, null=False)
    nome = models.CharField(max_length=50, blank=False, null=False)
    nome_de_guerra = models.CharField(max_length=20, blank=False, null=False)
    nasc = models.DateField(blank=False, null=False)
    cpf = models.CharField(max_length=14, blank=False, null=False, unique=True)
    email = models.EmailField(max_length=100, unique=True, blank=False, null=False)

    def __str__(self):
        return f'{self.re} {self.dig} {self.nome_de_guerra}'

    class Meta:
        ordering = ('re',)

    Migrações (Migrations)

    O Django utiliza um sistema de migrações para gerenciar as alterações no esquema do banco de dados de forma versionada e controlada.

    • makemigrations: Este comando inspeciona as alterações nos seus arquivos models.py e cria um novo arquivo de migração no diretório migrations/ do respectivo app. 

      python manage.py makemigrations <nome_do_app>

    • migrate: Este comando aplica as migrações pendentes ao banco de dados, alterando as tabelas, colunas, etc. 

      python manage.py migrate

    Esse fluxo de trabalho garante que as alterações no banco de dados sejam consistentes e aplicáveis em qualquer ambiente.
