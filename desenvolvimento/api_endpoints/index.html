    API Endpoints e Comunicação AJAX

    Embora o SisCoE seja uma aplicação majoritariamente renderizada no servidor, ele utiliza chamadas AJAX (Asynchronous JavaScript and XML) para diversas funcionalidades dinâmicas, como atualizações parciais de página, validações e interações em tempo real.

    Esses endpoints não seguem uma arquitetura REST estrita, mas funcionam como uma API interna, geralmente retornando respostas em formato JSON.

    Padrão de Resposta

    A maioria dos endpoints que respondem a chamadas AJAX retorna um objeto JSON com uma estrutura semelhante a esta:

    {
  "success": true,
  "message": "Operação realizada com sucesso!",
  "data": { ... }
}
    - success: Um booleano indicando se a operação foi bem-sucedida. - message: Uma mensagem descritiva para o usuário. - data: Um objeto contendo os dados solicitados (em caso de GET) ou dados atualizados (em caso de POST).

    Endpoints por App

    Abaixo estão alguns dos principais endpoints AJAX encontrados no sistema.

    App agenda

    Utilizado para a funcionalidade de calendário interativo.

    • POST /agenda/criar_lembrete/: Cria um novo lembrete para o usuário logado.
    • POST /agenda/criar_tarefa/: Cria uma nova tarefa.
    • GET /agenda/get_eventos/: Retorna uma lista de todos os eventos (lembretes e tarefas) do usuário em formato JSON para serem exibidos no FullCalendar.
    • DELETE /agenda/excluir_lembrete/<int:pk>/: Exclui um lembrete.

    App efetivo

    Contém vários endpoints para manipulação de dados do efetivo de forma dinâmica.

    • POST /efetivo/atualizar_imagem_perfil/<int:cadastro_id>/: Realiza o upload e a atualização da foto de perfil de um militar.
    • GET /efetivo/check_re_existente/: Verifica se um determinado RE (Registro de Efetivo) já existe no banco de dados. Usado para validação em formulários.
    • POST /efetivo/atualizar_categoria/<int:pk>/: Atualiza a categoria de um militar (ativo, inativo, licença, etc.).

    App lp (Licença Prêmio)

    Endpoints relacionados ao cálculo e visualização de Licença Prêmio.

    • GET /lp/calcular_lp/: Endpoint que recebe dados de um militar e retorna o cálculo de sua Licença Prêmio.
    • GET /lp/get_militar_data/: Busca dados de um militar específico para preencher formulários dinamicamente.

    App cursos

    Endpoints para gerenciar cursos e medalhas.

    • GET /cursos/api/cursos/: Retorna uma lista de cursos para um determinado militar.
    • POST /cursos/api/medalha/adicionar/: Adiciona uma nova medalha a um militar.
    • DELETE /cursos/api/medalha/excluir/<int:pk>/: Exclui uma medalha.

    Conclusão

    A aplicação faz uso extensivo de JsonResponse para criar uma experiência de usuário mais rica e dinâmica, evitando recarregamentos completos de página para muitas operações. Ao desenvolver novas funcionalidades, o padrão de usar views que respondem a requisições AJAX com JSON deve ser mantido para consistência.
