    Desenvolvimento Backend

    O backend do SisCoE é uma aplicação monolítica construída com o framework Django, escrito em Python. Ele é responsável por toda a lógica de negócio, processamento de dados, autenticação e por servir os templates do frontend.

    Framework Django

    O projeto utiliza o Django e segue sua arquitetura MVT (Model-View-Template):

    • Models (models.py): Definem a estrutura dos dados e a relação entre eles. São a única fonte de verdade sobre os dados do sistema, mapeando diretamente para as tabelas do banco de dados.
    • Views (views.py): Contêm a lógica que processa as requisições HTTP. As views recebem a requisição, interagem com os models para buscar ou manipular dados, e renderizam um template como resposta.
    • Templates (templates/): Camada de apresentação, responsável por exibir os dados para o usuário.

    Estrutura de Apps

    O backend é dividido em múltiplos "apps" Django, cada um encapsulando uma funcionalidade de negócio. Esta abordagem, conhecida como "pluggable apps", é uma das forças do Django e permite uma organização de código limpa e modular.

    Os principais apps e suas responsabilidades estão descritos na seção Componentes do Sistema.

    Principais Bibliotecas e Dependências

    O arquivo requirements.txt lista todas as dependências do backend. Algumas das mais importantes são:

    • Django: O framework web principal.
    • psycopg2-binary: Adaptador do banco de dados PostgreSQL para Python, permitindo a comunicação entre o Django e o banco.
    • gunicorn: Servidor de aplicação WSGI, usado para rodar a aplicação em produção.
    • python-decouple: Usado para separar as configurações do código, permitindo carregar variáveis de ambiente a partir de arquivos .env.
    • argon2-cffi: Biblioteca para hashing de senhas. O Argon2 é o algoritmo de hashing padrão recomendado pelo Django para segurança de senhas.
    • django-import-export: Facilita a importação e exportação de dados em formatos como CSV, XLS, etc., muito utilizado na interface de administração do Django.

    Utilitário manage.py

    O manage.py é um script de linha de comando que serve como o ponto de entrada para interagir com a aplicação Django.

    Comandos mais comuns:

    • python manage.py runserver: Inicia o servidor de desenvolvimento.
    • python manage.py migrate: Aplica as migrações do banco de dados.
    • python manage.py makemigrations: Cria novos arquivos de migração baseados nas alterações feitas nos models.py.
    • python manage.py createsuperuser: Cria um usuário administrador.
    • python manage.py shell: Abre um shell interativo do Python com o ambiente do Django carregado, útil para debugging e testes.
