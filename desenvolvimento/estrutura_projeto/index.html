    Estrutura do Projeto

    O repositório do SisCoE é organizado para separar claramente as diferentes partes do sistema, como o código do backend, a documentação e as configurações de deploy.

    Estrutura de Diretórios Principal

    /
├── backend/             # Contém todo o código-fonte do projeto Django.
├── contrib/             # Scripts e utilitários de apoio ao desenvolvimento.
├── docs/                # Arquivos da documentação técnica (gerada com MkDocs).
├── docker/              # Configurações para os contêineres Docker (ex: Nginx).
├── import/              # Scripts e arquivos para importação de dados.
├── node_modules/        # Dependências do Node.js (gerenciadas pelo npm).
├── venv/                # Ambiente virtual do Python (não versionado).
│
├── .gitignore           # Arquivos e pastas ignorados pelo Git.
├── docker-compose.yml   # Orquestração dos serviços Docker.
├── Dockerfile           # Define a imagem Docker para a aplicação Django.
├── manage.py            # Utilitário de linha de comando do Django.
├── mkdocs.yml           # Arquivo de configuração do MkDocs.
├── package.json         # Define as dependências de frontend (TailwindCSS).
├── requirements.txt     # Dependências do backend Python.
└── settings.py          # Arquivo principal de configuração do Django.

    Detalhamento

    Diretório backend/

    Este é o diretório mais importante, contendo o projeto Django.

    • backend/settings/: Embora exista, o arquivo principal de settings (settings.py) está na raiz. Este diretório pode ser para configurações mais específicas ou um artefato de uma estrutura anterior.
    • backend/urls.py: Arquivo de roteamento principal do Django.
    • backend/wsgi.py & asgi.py: Pontos de entrada para os servidores WSGI e ASGI.
    • backend/<app_name>/: Cada subdiretório aqui é um "app" Django, representando um módulo do sistema (ex: accounts, efetivo, core).
      • models.py: Define os modelos de dados (tabelas do banco).
      • views.py: Contém a lógica de negócio que processa as requisições.
      • urls.py: Define as rotas específicas do app.
      • forms.py: Define os formulários Django.
      • admin.py: Configura a interface de administração do Django para os modelos do app.
      • templates/: Contém os templates HTML do app.
      • migrations/: Armazena os arquivos de migração do banco de dados gerados pelo Django.

    Frontend (package.json, tailwind.config.js)

    O frontend não é uma SPA (Single Page Application) separada. Ele é construído com templates Django tradicionais. O package.json e o tailwind.config.js são utilizados como ferramentas de desenvolvimento para compilar o TailwindCSS, que é o framework CSS utilizado para a estilização da interface.

    Docker (Dockerfile, docker-compose.yml, docker/)

    • Dockerfile: Define a receita para construir a imagem Docker da aplicação Django, instalando as dependências e configurando o ambiente.
    • docker-compose.yml: Orquestra a execução de todos os serviços necessários para a aplicação rodar: a aplicação Django (app), o banco de dados (db), o pgadmin e o nginx como proxy reverso.
    • docker/config/: Contém arquivos de configuração que são montados nos contêineres, como a configuração do Nginx.

    Outros Diretórios

    • contrib/: Contém scripts auxiliares, como env_gen.py para gerar arquivos de ambiente.
    • import/: Contém scripts e planilhas para realizar importações de dados em massa para o sistema.
