    Deploy com Docker

    A containerização é uma parte central da estratégia de deploy e do ambiente de desenvolvimento do SisCoE. O projeto utiliza Docker e Docker Compose para garantir consistência, portabilidade e isolamento dos serviços.

    Dockerfile

    O Dockerfile na raiz do projeto contém a "receita" para construir a imagem da aplicação Django. Suas principais responsabilidades são:

    1. Imagem Base: Utiliza uma imagem base do Python.
    2. Copia o Código: Copia o código-fonte da aplicação para dentro da imagem.
    3. Instala Dependências: Instala as dependências Python listadas no requirements.txt.
    4. Expõe a Porta: Expõe a porta em que a aplicação Django será executada (geralmente a porta 8000).
    5. Comando de Execução: Define o comando padrão para iniciar a aplicação quando um contêiner for criado a partir da imagem (usando gunicorn).

    docker-compose.yml

    Este arquivo orquestra a execução de todos os contêineres que compõem o sistema. Ele define os seguintes serviços:

    • db:

      • Imagem: postgres:17-alpine
      • Responsabilidade: Contêiner do banco de dados PostgreSQL.
      • Volumes: Utiliza um volume nomeado (pgdata) para persistir os dados do banco de dados, garantindo que eles não sejam perdidos quando o contêiner é recriado.
      • Rede: Conectado à rede siscoe-network.

    • app:

      • Build: Constrói a imagem a partir do Dockerfile local.
      • Responsabilidade: O contêiner da aplicação Django.
      • Comando: Inicia o servidor de aplicação gunicorn.
      • Dependências: Depende do serviço db, garantindo que o banco de dados seja iniciado antes da aplicação.
      • Rede: Conectado à rede siscoe-network.

    • nginx:

      • Imagem: nginx
      • Responsabilidade: Atua como um proxy reverso. Ele recebe as requisições HTTP na porta 80 e as encaminha para a aplicação Django na porta 8000. Também é responsável por servir os arquivos estáticos de forma eficiente.
      • Volumes: Monta o arquivo de configuração do Nginx de docker/config/nginx/ para dentro do contêiner.
      • Dependências: Depende do serviço app.
      • Rede: Conectado à rede siscoe-network.

    • pgadmin:

      • Imagem: dpage/pgadmin4
      • Responsabilidade: Fornece uma interface web para administrar o banco de dados PostgreSQL, útil para desenvolvimento e debugging.

    • mailhog:

      • Imagem: mailhog/mailhog
      • Responsabilidade: Um servidor de e-mail falso que captura todos os e-mails enviados pela aplicação Django durante o desenvolvimento, permitindo visualizá-los em uma interface web sem enviá-los para destinatários reais.

    Como Executar com Docker

    Construir e iniciar os contêineres: 

    docker-compose up --build -d
    - --build: Força a reconstrução da imagem da aplicação se houver alterações no Dockerfile ou no código-fonte. - -d: Executa os contêineres em modo "detached" (em segundo plano).

    Parar os contêineres: 

    docker-compose down

    Visualizar os logs: 

    docker-compose logs -f <nome_do_servico>
# Exemplo:
docker-compose logs -f app
