    Logs e Monitoramento

    A observabilidade é crucial para manter a saúde e a performance da aplicação em produção. Esta seção descreve a configuração de logs atual e sugere melhorias para um monitoramento mais robusto.

    Configuração de Logs do Django

    O SisCoE possui uma configuração de logging no arquivo settings.py, que direciona os logs para dois locais:

    1. Console: Logs são enviados para a saída padrão (stdout), o que permite que sejam facilmente capturados pelo Docker.
    2. Arquivo: Logs de nível DEBUG e superiores são salvos no arquivo debug.log na raiz do projeto.

    Trecho da configuração (settings.py): 

    LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'formatters': {
        'verbose': {
            'format': '{levelname} {asctime} {module} {message}',
            'style': '{',
        },
    },
    'handlers': {
        'console': {
            'class': 'logging.StreamHandler',
        },
        'file': {
            'level': 'DEBUG',
            'class': 'logging.FileHandler',
            'filename': 'debug.log',
            'formatter': 'verbose'
        },
    },
    'loggers': {
        'django': {
            'handlers': ['console', 'file'],
            'level': 'INFO',
            'propagate': False,
        },
    }
}

    Acessando Logs via Docker

    Quando a aplicação está rodando via Docker Compose, a maneira mais fácil de visualizar os logs em tempo real é através do comando:

    # Para visualizar os logs do contêiner da aplicação Django
docker-compose logs -f app

# Para visualizar os logs do Nginx
docker-compose logs -f nginx

    Recomendações para Monitoramento

    A configuração atual é adequada para desenvolvimento, mas para um ambiente de produção, um sistema de monitoramento mais completo é recomendado.

    1. Logging Estruturado

    Em vez de logs em texto plano, adote o logging estruturado (ex: JSON). Isso facilita a busca, a filtragem e a análise dos logs em ferramentas centralizadas. Bibliotecas como python-json-logger podem ser integradas ao Django.

    2. Centralização de Logs

    Utilize uma stack de centralização de logs como o Loki com Grafana ou o ELK Stack (Elasticsearch, Logstash, Kibana). Os contêineres seriam configurados para enviar seus logs para um desses sistemas, permitindo a visualização e a criação de alertas em um único dashboard.

    3. Monitoramento de Métricas com Prometheus e Grafana

    • Prometheus: Uma ferramenta de monitoramento que coleta métricas da aplicação em intervalos de tempo. Uma biblioteca como django-prometheus pode ser usada para expor métricas da aplicação Django (ex: número de requisições, tempo de resposta, erros 5xx).
    • Grafana: Uma ferramenta de visualização que se conecta ao Prometheus para criar dashboards e alertas. Com o Grafana, seria possível monitorar a saúde da aplicação, o consumo de recursos dos contêineres e o tráfego de rede em tempo real.

    4. Rastreamento de Erros (Error Tracking)

    Integre um serviço de rastreamento de erros como o Sentry ou o GlitchTip. Essas ferramentas capturam exceções não tratadas na aplicação em tempo real, agrupam-nas, e notificam a equipe de desenvolvimento com o stack trace completo, facilitando a depuração de problemas em produção.
